Wordpress Search

Wordpress Search Functionality

WordPress has a built in search functionality allowing anyone to find content inside a site. By default it searches through post and page titles, post and page content, excerpts, and custom posts and content. It will not search for comments, tag names, or category names. You are able to add that through plugins though.

Searches are done though an SQL query based off of the search terms. Once a search has been performed the results get displayed by using your theme's search results template, in our case search.php.

The Role of URI

URI stands for Uniform Resource Identifier. Which is a string of characters identifying things like a website or file. WordPress has a default structure of /?s=search+term. The term within the URI can be used to highlight matching content or display the search term within the search form. This format also is less attractive to search engine crawlers.

Current/Active Search Terms

You can show the current or active search term in the search form field after the search is performed. You do this by adding a value attribute to the search input field that echoes the current search term using PHP. Example:

<input type="search" name="s" value="<?php echo get_search_query(); ?>" placeholder="Search...">

This gets the search term using WordPress's get_search_query() function and displays it in the search form, which lets people see and modify their current search.

Summary

WordPress search functionality is a great feature that allows site visitors to find content within your site. The search query is show in the URI which WordPress uses to generate results. While the default search is good you can use filters or plugins to expand your needs.
