Wordpress Template Hierarchy

What is the Template Hierarchy System?

This hierarchy system determines which template file is going to be used to display a specific page on your WordPress site. This gives a structure for WordPress to pick which template file when rendering different types of content.

Template Order

The order wordpress follows when selecting a template file is:

  1. First it looks for template files that match a specific or current page.
  2. Then, if it does not find specific template it moves on to general templates you may have.
  3. Lastly, if no matching templates are found WordPress will fall back to the most generic or default template it has available to it.

For example I used the Donate and Donar Recognition page for the assignment so it would look like this:

  1. Custom template file: page_full-width.php
  2. page-{donate}.php
  3. page-{id}.php (For Donor Recognition it would be 90)
  4. page.php
  5. single.php
  6. index.php

File Naming Conventions

Some examples would be:

  • Author templates: author-{trevor}.php or author-{id}
  • General templates: index.php, archive.php, single.php, page.php
  • Category templates: category-{slug}.php and category-{id}.php
  • Date templates: date.php, month.php, day.php, year.php

ID vs. Slug

The difference between ID and Slug is that ID is a numeric identifier so a page template would be page-23.php. While a Slug is a text-based identifier so in my case I did page-donate.php. Wordpress prioritizes slug based filenames over ID base ones which I think adds more readability.

Summary

With template hierarchy its a "find the right file" thing. We could have many templates that alter how our sites look so WordPress goes through that checklist or hieararchy to figure out which template to use. It starts of specific then goes down to general. WordPress prefers word versions because it is easier for humans to understand. Without a matching template it will use the default.
